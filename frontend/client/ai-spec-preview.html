HTML Output (Phase 6.1 — Layout Generator HTML path)

Self-contained HTML the Layout Generator emitted. When RENDER_USE_HTML=true, the renderer uses this directly via page.setContent. Iframe below renders it sandboxed at native viewport; source toggle reveals the raw HTML. Palette swatches show the colors the LLM picked.

HTML Validation (Phase 6.2 — Pre-Judge filter input)

Deterministic checks against the winning HTML candidate. Hard violations drop candidates before the Judge sees them (parse failures, missing <html>/<body>, <script> tags, hero image 404s, proof-strategy unsupported). Soft warnings (contrast below AA, image probe failures, palette concerns) surface here; render still proceeds.

AI Image Reference (Phase X.1 shadow — gpt-image-1)

A "photoreal polish" of the design. Seeded from the HTML render when present (Phase 6.4 — refines the rendered HTML into a photoreal version); falls back to catalog hero or UGC source otherwise. Currently shadow only — not used as the production ad until the Campaign toggle is on.
